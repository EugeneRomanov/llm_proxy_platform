# LLM Proxy Platform — Агентная платформа с умной маршрутизацией

## Часть 1: Описание проекта и архитектура

---

## 1. Описание проекта

**LLM Proxy Platform** — это высоконагруженная прокси-платформа для маршрутизации запросов к LLM (Large Language Models) и A2A-агентам. Система обеспечивает интеллектуальное распределение трафика между несколькими провайдерами LLM, сбор метрик, мониторинг, защиту от вредоносных запросов и авторизацию.

### 1.1. Основные задачи платформы

| Задача | Описание |
|--------|----------|
| **Единая точка входа** | Унифицированный API для всех LLM-запросов |
| **Умная маршрутизация** | Выбор провайдера на основе латентности и health-статуса |
| **Автоматическое восстановление** | Блокировка проблемных провайдеров и их автоматическое восстановление |
| **Регистрация агентов** | Хранение и управление A2A-агентами с их Agent Cards |
| **Полная наблюдаемость** | Метрики, трейсинг и визуализация через Prometheus, Grafana, MLFlow |
| **Безопасность** | Guardrails для фильтрации вредоносных запросов + авторизация по токену |

### 1.2. Целевая аудитория

- **Разработчики A2A-агентов** — для регистрации и управления агентами
- **Команды MLOps** — для управления LLM-инфраструктурой
- **Компании с несколькими LLM-провайдерами** — для оптимизации затрат и отказоустойчивости

## 2. Архитектура

### 2.1. Компоненты системы

| Компонент | Описание | Технологии |
|-----------|----------|------------|
| **LLM Proxy** | Основной сервис, обрабатывающий запросы | FastAPI, Uvicorn, asyncio |
| **Smart Balancer** | Модуль умной маршрутизации с EMA-латентностью | Python |
| **Agent Registry** | Реестр A2A-агентов с CRUD операциями | SQLAlchemy, SQLite |
| **Guardrails** | Фильтр вредоносных запросов (prompt injection, SQL injection) | regex |
| **Mock LLM** | Тестовые LLM-провайдеры для нагрузки | FastAPI |
| **Prometheus** | Сбор метрик производительности | Prometheus |
| **Grafana** | Визуализация метрик и дашборды | Grafana |
| **MLFlow** | Трейсинг LLM-вызовов и логирование параметров | MLFlow |

### 2.2. Поток данных

1. **Входящий запрос** → клиент отправляет запрос к `/v1/chat/completions`
2. **Авторизация** → проверка `X-Agent-Token` заголовка
3. **Guardrails** → проверка содержимого на вредоносные паттерны
4. **Выбор провайдера** → Smart Balancer выбирает оптимального провайдера
5. **Проксирование** → запрос перенаправляется к выбранному провайдеру
6. **Стриминг ответа** → ответ возвращается клиенту в реальном времени
7. **Сбор метрик** → TTFT, латентность, токены, стоимость
8. **MLFlow трейсинг** → логирование параметров и метрик

## 3. Функциональные возможности

### 3.1. Level 1 — Базовый функционал (10 баллов)

| Функция | Описание | Статус |
|---------|----------|--------|
| **Docker Compose** | Полная контейнеризация всех сервисов | ✅ |
| **Mock LLM провайдеры** | 2 тестовых провайдера с симуляцией задержки 200ms | ✅ |
| **Прокси со стримингом** | Потоковая передача ответов, поддержка SSE | ✅ |
| **Round Robin балансировка** | Равномерное распределение запросов (в lvl1 версии) | ✅ |
| **OpenTelemetry** | Сбор метрик времени ответа | ✅ |
| **Prometheus + Grafana** | Экспорт метрик и визуализация | ✅ |
| **Health-check** | Эндпоинт `/health` для проверки состояния | ✅ |

### 3.2. Level 2 — Умная маршрутизация (20 баллов)

| Функция | Описание | Статус |
|---------|----------|--------|
| **Agent Registry** | CRUD операции для A2A-агентов | ✅ |
| **Динамическая регистрация** | Регистрация провайдеров с ценой, лимитами, таймаутами | ✅ |
| **Latency-based routing** | Выбор провайдера с минимальной EMA-латентностью (α=0.3) | ✅ |
| **Health-aware routing** | Блокировка провайдера после 3 ошибок на 30 секунд | ✅ |
| **TTFT метрики** | Время до первого токена (time-to-first-token) | ✅ |
| **Метрики стоимости** | Подсчет стоимости на основе цены за токен | ✅ |
| **MLFlow интеграция** | Трейсинг параметров и метрик LLM-вызовов | ✅ |

### 3.3. Level 3 — Продвинутая платформа (25 баллов)

| Функция | Описание | Статус |
|---------|----------|--------|
| **Guardrails** | Фильтрация prompt injection, API-ключей, SQL инъекций | ✅ |
| **Авторизация** | Проверка X-Agent-Token заголовка | ✅ |
| **Нагрузочное тестирование** | Скрипт с симуляцией отказов (20% ошибок) | ✅ |
| **Throughput анализ** | Расчет пропускной способности и P95 латентности | ✅ |
